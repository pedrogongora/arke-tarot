# Internationalization (i18n)

## Library: next-intl 3

- Locale lives in the URL segment: `/en/...`, `/es/...`
- `src/middleware.ts` redirects `/` and non-prefixed routes to the appropriate locale
- Server and Client Components both use `useTranslations()` or `getTranslations()`

## Files

| File | Purpose |
|---|---|
| `src/i18n/routing.ts` | Declares supported locales and default locale |
| `src/i18n/request.ts` | Server-side config: loads messages for the request locale |
| `src/middleware.ts` | next-intl middleware that handles locale routing |
| `messages/en.json` | English translations |
| `messages/es.json` | Spanish translations |

## Adding a New Language

1. Create `messages/[locale].json` with the same key structure as `en.json`
2. Add the locale to `src/i18n/routing.ts`:
   ```typescript
   export const routing = defineRouting({
     locales: ['en', 'es', 'fr'],  // add here
     defaultLocale: 'en'
   });
   ```
3. Add the locale to `UserSettings.locale` type in `src/types/index.ts`

## Using Translations in Components

**Server Components:**
```typescript
import { getTranslations } from 'next-intl/server';
const t = await getTranslations('spreads');
return <h1>{t('oneCard.name')}</h1>;
```

**Client Components:**
```typescript
'use client';
import { useTranslations } from 'next-intl';
const t = useTranslations('nav');
return <span>{t('home')}</span>;
```
